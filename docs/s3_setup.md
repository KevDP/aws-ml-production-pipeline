# S3 Storage Setup

This document describes the storage configuration used for the cloud-based machine learning pipeline.  
The system uses Amazon S3 as the central data storage layer for raw datasets, processed data, and trained models.

The storage layer enables scalable data ingestion and easy integration with machine learning services such as SageMaker.

---

# Bucket Structure

The project uses a single S3 bucket to organize all data assets required for the pipeline.

Bucket name example:

- ml-demand-forecasting-kevin

Directory structure:

- ml-demand-forecasting-kevin/ 
    - raw/
        - demand_data.csv
    - processed/
    - models
        - model.joblib

### Folder descriptions

*raw/*  
Contains the original dataset used for model training.  
This data is uploaded directly to S3 and remains unchanged.

*processed/*  
This folder stores cleaned and transformed datasets generated during feature engineering steps.

*models/*  
Stores trained machine learning model artifacts generated by the training pipeline.

---

# Dataset Description

The dataset used in this project simulates retail demand forecasting scenarios.

The goal of the dataset is to predict product demand based on historical sales and contextual features.

### Dataset Characteristics

- Synthetic dataset generated for demonstration purposes
- Simulates retail demand behavior
- Includes temporal and contextual features

### Main Features

| Column        | Description                                       |
|---------------|---------------------------------------------------|
| date          | Date of the observation                           |
| store_id      | Unique identifier for the store                   |
| product_id    | Unique identifier for the product                 |
| price         | Product price at the given time                   |
| promotion     | Binary flag indicating promotional campaigns      |
| holiday       | Binary flag indicating holidays                   |
| temperature   | Environmental feature that may influence sales    |
| weekly_sales  | Target variable representing product demand       |

### Dataset Size

- Approximately 100,000 rows
- Simulates multiple stores and products across several years
- Includes seasonality and promotional patterns

# Data Flow

The data pipeline follows this flow:

1. Dataset is uploaded to the *raw/* folder in Amazon S3
2. SageMaker training jobs load the dataset directly from S3
3. Feature engineering generates transformed datasets stored in *processed/*
4. Trained models are saved in *models/*

This structure allows scalable machine learning workflows while keeping raw and processed data clearly separated.

---

# Design Considerations

The storage design follows common data engineering practices:

- Separation between raw and processed data
- Versionable model artifacts
- Scalable cloud-native storage

Using S3 also enables easy integration with services such as:

- AWS SageMaker training jobs
- serverless inference workflows
- automated ML pipelines
